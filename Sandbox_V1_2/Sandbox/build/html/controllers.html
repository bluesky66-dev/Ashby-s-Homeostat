Controllers¶

Controller class¶

class Sandbox.Controller(inputs_n: int, commands_n: int, step_fun: Callable[[float, List[float], List[float], List[float]], List[float]], noisemakers: Optional[List[Sandbox.noise.NoiseSource]] = None, noisemakers_inds=None, params: Optional[List[float]] = None, adapt_fun: Optional[Callable[[List[float], List[float], List[float]], None]] = None, adapt_enabled: bool = True, test_interval: float = 0)[source]¶

This class represents a controller for an Agent. There are two routes to customising Controller or one of its subclasses:

  1. Using step_fun(dt, inputs, params, state). step_fun is a function which can be written anywhere in the code for your experiment and passed to a Controller when it is constructed. This will only work for relatively simple controllers.

  2. By creating a subclass of Controller, like BraitenbergController, where you can add any attributes you need to by writing a new __init__ method (which should call super().__init__, so that attributes from Controller are inherited and set up correctly). In some cases, you may also need to override the step method of Controller (possibly using super().step to inherit existing functionality), and to add other methods to the class.

If you want to create a self-adaptive controller, then you can either create a subclass of Controller or - for relatively simple cases - you can implement an adapt_fun(dt, inputs_hist, commands_hist, params_hist) function and pass it into a Controller when it is constructed.

__init__(inputs_n: int, commands_n: int, step_fun: Callable[[float, List[float], List[float], List[float]], List[float]], noisemakers: List[NoiseSource] = None, noisemakers_inds=None, params: List[float] = None, adapt_fun: Callable[[List[float], List[float], List[float]], None] = None, adapt_enabled: bool = True, test_interval: float = 0)[source]¶
Parameters

  • inputs_n (int) – The number of inputs expected by the controller.

  • commands_n (int) – The number of outputs (commands) that the controller is expected to return from its step method.

  • step_fun (function) – The function which will be used to generate the controller’s outputs, given the inputs to the controller’s step method, the interval of time to integrate over, and any state and parameters the controller makes use of.

  • noisemakers (List of NoiseSource objects.) – A list of noise sources which will potentially affect the commands the controller outputs.

  • noisemaker_inds (List of integers.) – A list of indices of commands which will potentially have noise added to them. Any indices which are out of range in either the list of noisemakers or commands will be ignored.

  • params (list of floats.) – A list of parameters used by the controller. These parameters will be used in the controller’s step_fun function, which will be caused from the controller’s step method.

  • adapt_fun (function) – A function wich can be used to adapt a controller, by changing its parameters. It only has access to the same data that the controller does: its histories of inputs, outputs, and parameter values.

  • adapt_enabled (bool) – When a Controller has an adapt_fun, that function will only be used when adapt_enabled is set to True.

  • test_interval (float) – The period of time to wait between parameter changes, if an adapt_fun is being used.

reset() → None[source]¶

A method to reset the controller to its initial state, so that it can be reused without the data and states from a previous simulation run affecting the next run.

You will typically want to get the controller’s data before resetting it, e.g. so that you can store if for the purposes of analysis. The most convenient way to do this will often be to call the get_data_and_reset method defined in the System class.

step(dt: float, inputs: List[float], state: List[float] = []) → List[float][source]¶

A method to step a controller forwards in time.

Parameters

  • dt (float) – The interval of time to integrate the controller over.

  • inputs (list of floats) – The inputs to the controller.

  • state (list of floats) – This variable can be used to pass in information about an agent’s state (e.g. for a robot this could be [x, y, theta]). NOTE: even though I have included this shortcut, its use should normally be avoided - strictly speaking, an agent should use its sensors to estimate its state, and then pass its estimates to its controller in inputs.

Returns

List of commands.

Return type

list of floats.

BraitenbergController class¶

class Sandbox.BraitenbergController(step_fun: Callable[[float, List[float], List[float], List[float]], List[float]], left_noisemaker: Optional[Sandbox.noise.NoiseSource] = None, right_noisemaker: Optional[Sandbox.noise.NoiseSource] = None, gain=1)[source]¶

A subclass of Controller, which can be used to control an instance of Robot. It, optionally, has a noisemaker for each of the control commands to the robots two motors. See the BraitenbergController.py source code file for examples of controllers which implement the behaviours of simple Braitenberg Vehicles.

__init__(step_fun: Callable[[float, List[float], List[float], List[float]], List[float]], left_noisemaker: NoiseSource = None, right_noisemaker: NoiseSource = None, gain=1)[source]¶
Parameters

  • step_fun (function) – The function which will be used to generate the controller’s outputs, given the inputs to the controller’s step method, the interval of time to integrate over, and any state and parameters the controller makes use of.

  • left_noisemaker (NoiseSource) – A noise source which will potentially affect the command which the controller outputs for the robot’s left motor.

  • right_noisemaker (NoiseSource) – A noise source which will potentially affect the command which the controller outputs for the robot’s right motor.

  • gain (float) – A gain parameter which can be used to control how fast the robot moves in response to being stimulated by a light source.

Light seeking control function¶

Sandbox.light_seeking(dt: float, inputs: List[float], params: List[float], state: List[float] = []) → List[float][source]¶

A function, for use with a BraitenbergController, which implements light-seeking behaviour.

Parameters

  • dt (float) – Interval of time to integrate the controller over.

  • inputs (list[float]) – The list of sensory inputs to the controller.

  • params (list[float]) – The list of controller parameters (in this case, just the controller’s gain attribute).

  • state (list[float]) – This variable is included to allow elements of an agent’s state to be used in its controller - in general, use of this is discouraged - where an agent’s state is to be used in control, it should either be measured by an appropriate sensor, and therefore be part of the inputs vector, or it should be inferred by the controller itself.

Returns

List of motor commands.

Return type

list of floats.
