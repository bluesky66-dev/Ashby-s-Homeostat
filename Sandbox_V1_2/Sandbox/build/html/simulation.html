Simulator class¶

class Sandbox.Simulator(agents: List[Sandbox.Agent.Agent], envs: List[Sandbox.System.System], duration: float, dt: float, obj_fun=None, disturbances: List[Sandbox.DisturbanceSource.DisturbanceSource] = [])[source]¶

A class for running a Sandbox simulation.

It is possible to write your own simulation loop, without too much difficulty, so this class really exists for convenience, for standardising, and to cut down on the amount of code in main scripts.

__init__(agents: List[Agent], envs: List[System], duration: float, dt: float, obj_fun=None, disturbances: List[DisturbanceSource] = [])[source]¶
Parameters

  • agents (List[Agent]) – The list of agents to simulate.

  • envs (List[System]) – The list of systems which are not agents or disturbances to simulate.

  • duration (float) – The length of simulation time to run the simulation for.

  • dt (float) – The interval of time to integrate systems over.

  • obj_fun (function) – An objective function for evaluating the performance of some aspect of the simulation. Not used in the current implementation - it has been temporarily removed from all methods, and will either be introduced later or completely removed.

  • disturbances (List[DisturbanceSource]) – The list of disturbances to simulate.

Animator class¶

class Sandbox.Animator(systems: Optional[List[Sandbox.System.System]] = None, screen_width: int = 700, paused: bool = False, delay: int = 0)[source]¶

An class for animating a simulation using PyGame. To simplify scaling, the display is square, i.e. has equal width and height. Whenever a system moves beyond any edge of the currently displayed area, the view will zoom out accordingly, byt the same amount in both axes. The view will not zoom in again if the system moves back towards the centre.

__init__(systems: List[System] = None, screen_width: float = 700, paused: bool = False, delay: float = 0)[source]¶
Parameters

  • systems (List[System]) – A list of systems to draw on the PyGame display.

  • screen_width (int) – The PyGame display height and width, in pixels.

  • paused (bool) – A flag, set by keyboard inputs in the animation, to specify whether or not the animation, and underlying simulation, should be paused. It is the job of the simulation to actually implement that pause, e.g. see the run_once method of SimulationRunner

  • delay (int) – The number of 10s of milliseconds to delay the animation by inbetween each pair of successive simulation steps. Can be used to slow an animation/simulation down in order to see what is happening more clearly.

SimulationRunner class¶

class Sandbox.SimulationRunner(sim: Sandbox.Simulator.Simulator, animate: bool = False, screen_width: int = 700, pause_ani=False, animation_delay: int = 0, perturb: bool = True)[source]¶

A class for running a batch of simulations, which may or may not be animated, as required.

__init__(sim: Simulator, animate: bool = False, screen_width: int = 700, pause_ani=False, animation_delay: int = 0)[source]¶
Parameters

  • sim (Simulator) – The Simulator which will be used (note: future implementations will not need this to be passed in, as it can easily be constructed here).

  • animate (bool) – A flag which determines whether or not the simulation runs will be animated. Note: if this is set to False, then no Animator will be constructed, so it cannot be changed during simulation runs. When this flag is set to True, the Animator is constructed, and you can the choose to use it, or skip it, using its controls.

  • screen_width (int) – The PyGame display height and width, in pixels.

  • pause_ani (bool) – If this flag is set to True (which is it by default), and simulation runs are being animated, then animations will always begin in the paused state.

  • delay (int) – The number of 10s of milliseconds to delay the animation by, when one is used, inbetween each pair of successive simulation steps. Can be used to slow an animation/simulation down in order to see what is happening more clearly.
